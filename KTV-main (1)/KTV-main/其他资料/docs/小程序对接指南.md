## 2. 认证机制

### 2.1 流程

```
用户登录 → 后端验证 → 返回 JWT Token
    ↓
前端存储 Token（小程序用 wx.setStorageSync）
    ↓
后续请求 Header 携带: Authorization: Bearer <token>
    ↓
Token 过期（默认 1440 分钟 = 24 小时）→ 需重新登录
```

### 2.2 Token 格式

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

### 2.3 需要认证的接口

所有接口都需要 Token，除了：
- `POST /api/auth/login`
- `POST /api/auth/register`

## 3. 完整 API 接口列表

> 以下所有请求/响应均为 JSON 格式。`[Auth]` 标记的接口需要携带 JWT Token。

### 3.1 认证模块

#### 登录
```
POST /api/auth/login
Content-Type: application/json

请求体：
{
  "username": "string",  // 用户名/手机号/邮箱（任一即可）
  "password": "string"
}

成功响应 200：
{
  "token": "eyJhbGciOi...",
  "user": {
    "id": 1,
    "username": "user_001",
    "displayName": "张三",
    "phone": "13800001111",
    "email": null,
    "avatarUrl": "/uploads/avatars/xxx.jpg",
    "status": "active",
    "createdAt": "2023-10-25T10:00:00"
  }
}

失败响应 401：
{ "message": "用户名或密码错误" }
```

#### 注册
```
POST /api/auth/register
Content-Type: application/json

请求体：
{
  "username": "string",    // 可选，至少填 username/phone/email 其中之一
  "phone": "string",       // 可选
  "email": "string",       // 可选
  "password": "string",    // 必填
  "displayName": "string"  // 必填（昵称）
}

成功响应 200：
{ "id": 10, "message": "注册成功" }

失败响应 400：
{ "message": "用户名已存在" }
```

#### 退出登录
```
POST /api/auth/logout
[Auth]

成功响应 200：（无返回体）
```

### 3.4 播放同步模块

> 所有房间成员通过 2 秒轮询获取播放状态，实现播放同步。只有当前歌曲的**点歌人**（歌曲主）可以控制播放。

#### 获取播放状态
```
GET /api/room/playback
[Auth]

成功响应 200：
{
  "hasTrack": true,
  "currentQueueItemId": 108,
  "songId": 42,
  "title": "晴天",
  "artist": "周杰伦",
  "coverUrl": "/uploads/covers/xxx.jpg",
  "mediaUrl": "/uploads/music/xxx.mp3",
  "lrcUrl": "/uploads/lyrics/xxx.lrc",
  "orderedByUserId": 5,
  "orderedByName": "张三",
  "isPlaying": true,
  "currentTime": 35.2,
  "duration": 254,
  "playMode": "off",
  "hasPrev": true
}

// 无歌曲时：
{ "hasTrack": false }
```

- `currentTime`：服务端根据 `StartedAt` 实时计算，前端直接 seek 到该位置
- `playMode`：`off`(播完停止) | `repeat-all`(列表循环) | `repeat-one`(单曲循环) | `shuffle`(随机播放)
- `hasPrev`：history stack 是否有上一首
- 自动切歌：当 `currentTime >= duration` 时，下次请求会自动推进到下一首

#### 播放指定歌曲
```
POST /api/room/playback/play
[Auth]
Content-Type: application/json

{ "queueItemId": 108 }

成功响应 200
失败 403：{ "message": "只有当前歌曲的点歌人才能控制播放" }
```

#### 暂停
```
POST /api/room/playback/pause
[Auth]

成功响应 200
失败 403：权限不足
```

#### 继续播放
```
POST /api/room/playback/resume
[Auth]

成功响应 200
失败 403：权限不足
```

#### 进度跳转
```
POST /api/room/playback/seek
[Auth]
Content-Type: application/json

{ "position": 120.5 }

成功响应 200
失败 403：权限不足
```

#### 下一首
```
POST /api/room/playback/next
[Auth]

成功响应 200
- repeat-one：重播当前
- shuffle：随机选一首
- repeat-all：末尾循环回开头
- off：末尾停止播放
失败 403：权限不足
```

#### 上一首
```
POST /api/room/playback/prev
[Auth]

成功响应 200
失败 400：{ "message": "没有上一首" }
失败 403：权限不足

// 从 history stack 弹出，与 queue 解耦
// Play 自动 push 当前到 history，支持链式回退
```

#### 切换播放模式
```
POST /api/room/playback/mode
[Auth]
Content-Type: application/json

{ "mode": "repeat-all" }

// mode: off | repeat-all | repeat-one | shuffle
成功响应 200
失败 403：权限不足
```

#### 权限规则

| 操作 | 谁能做 |
|------|--------|
| play（开始播放） | 任何人（无歌曲时）或当前歌曲的点歌人 |
| pause/resume/seek/next/prev/mode | 仅当前歌曲的**点歌人**（`orderedByUserId`） |

播放控制权随歌曲切换自动转移：A 点的歌播完切到 B 点的歌 → B 获得控制权。

## 4. 数据同步机制

小程序端需要自己实现实时同步，和 Web 端一样的方案。

### 4.1 轮询方案

| 数据 | 接口 | 建议轮询间隔 |
|------|------|------------|
| 播放队列 | `GET /api/room/queue` | 2 秒 |
| 聊天消息 | `GET /api/chat/messages?roomId=X` | 2 秒 |
| 房间信息 | `GET /api/room/current` | 2 秒 |

### 4.2 关键代码模式（参考 Web 端实现）

```javascript
// 页面 onLoad 时启动轮询，onUnload 时清除
onLoad() {
  this.loadData()
  this.pollTimer = setInterval(() => {
    this.loadQueue()
    this.loadChatMessages()
    this.loadRoomInfo()
  }, 2000)
},
onUnload() {
  clearInterval(this.pollTimer)
}
```

### 4.3 同步原理

```
用户 A 点歌 → POST /api/room/queue → 写入 PlayQueue 表
    ↓
用户 B 的下次轮询（最多 2 秒后）
    ↓
GET /api/room/queue → 查 PlayQueue 表 → 返回最新队列
    ↓
前端更新 UI
```

聊天同理，只是写入/读取 ChatMessages 表。

## 7. 已知限制和注意事项

1. **Token 过期**：默认 24 小时，过期后需重新登录。小程序端应在请求返回 401 时跳转登录页。
2. **同一用户只能在一个房间**：再次加入其他房间会自动退出原房间。
3. **歌曲去重**：同一房间内同一首歌只能被点一次（未播放状态）。
4. **消息长度**：聊天消息最多 200 字符，含敏感词会自动过滤（替换为 `*`）。
5. **轮询消耗**：每个用户每秒 1 次请求（聊天 + 队列各 2 秒一次），当前用户量小无所谓。
6. **密码未哈希**：课程项目，密码明文存储，不上生产。
7. **`GET /api/auth/me`** 未完善，用户信息建议从登录响应缓存。

### [2026-05-24] v2.0.8 — 播放同步模块（歌曲主模式 + 播放模式）

#### 新增接口

共 7 个播放控制 API（见 3.4 播放同步模块）：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/room/playback` | GET | 获取播放状态（2s 轮询） |
| `/api/room/playback/play` | POST | 播放指定队列项 |
| `/api/room/playback/pause` | POST | 暂停 |
| `/api/room/playback/resume` | POST | 继续播放 |
| `/api/room/playback/seek` | POST | 进度跳转 |
| `/api/room/playback/next` | POST | 下一首 |
| `/api/room/playback/prev` | POST | 上一首 |
| `/api/room/playback/mode` | POST | 切换播放模式 |

#### 核心架构

- **队列（queue）**：只表示待播放列表，reorder 不影响播放历史
- **历史栈（history stack）**：记录真实播放路径，prev 从这里 pop，与 queue 完全解耦
- **当前指针（current pointer）**：当前播放的歌曲快照

#### 播放模式

| 模式 | 行为 |
|------|------|
| `off` | 播完即停 |
| `repeat-all` | 列表循环（末尾回到 queue[0]） |
| `repeat-one` | 单曲循环（不推入 history） |
| `shuffle` | 随机播放（排除当前） |

#### 响应变更

`GET /api/room/playback` 响应新增字段：`hasPrev`（boolean，history stack 是否有上一首）

#### 队列模块变更

`GET /api/room/queue` 响应中每个 item 新增字段：`orderedByUserId`（点歌人 ID）
