# 声域友 线上社交 KTV - 产品需求文档

> 版本：2.0.7
> 最后更新：2026-05-19
> 项目性质：Web 应用学科课程作业（团队项目）
> 产品方向：从线下 KTV 管理系统转型为线上社交 KTV 平台

---

## 1. 项目概述

### 1.1 产品名称
声域友（Hydro-Sonic Canvas / Azure Echo）

### 1.2 产品定位
一套**线上社交 KTV 平台**，用户通过 Web 浏览器创建或加入虚拟房间，在房间内与朋友一起点歌、排队、实时聊天、享受 KTV 体验。管理端提供房间监控、歌曲库维护、用户管理和反馈处理功能。

### 1.3 产品目标
- 将课程作业中的原型设计完整落地为可运行的全栈 Web 应用
- 管理端覆盖线上 KTV 运营所需的管理功能（房间监控、歌曲管理、用户管理、反馈处理）
- Web 端覆盖用户在虚拟房间内的核心使用场景（注册登录、房间加入、选歌、点歌、排队、聊天、收藏）

### 1.4 项目范围
- **包含**：管理端 6 个页面 + Web 端 6 个页面 + 后端 RESTful API + 数据库设计
- **不包含**：支付/计费系统、移动端适配、多租户 SaaS 架构、视频通话

---

## 2. 目标用户

### 2.1 管理端用户 — 系统管理员

| 属性 | 描述 |
|------|------|
| 角色 | 平台管理员（课程项目中由组员兼任） |
| 技术水平 | 会基本电脑操作，无需技术背景 |
| 使用场景 | 监控在线房间状态、维护歌曲库、管理用户账户、处理用户反馈、查看在线数据 |
| 核心痛点 | 需要一个统一后台管理线上 KTV 平台的各项运营数据 |

### 2.2 Web 端用户 — 线上 KTV 用户

| 属性 | 描述 |
|------|------|
| 角色 | 注册用户，与朋友一起在线唱歌 |
| 技术水平 | 普通手机/电脑用户 |
| 使用场景 | 注册登录后加入虚拟房间，浏览歌曲库、点歌排队、实时聊天、查看播放队列、管理个人收藏 |
| 核心痛点 | 想和朋友远程一起 KTV，但现有工具缺乏同步点歌、排队和实时互动的体验 |

## 3. 核心功能

### 3.1 管理端功能

#### 3.1.1 仪表盘

**功能描述**：管理员登录后看到的首页，展示平台实时运营数据概览。

**输入**：无（自动加载）

**输出**：
- 4 张统计卡片：在线房间数、在线用户数、今日创建房间数、总注册用户数
- 最新房间动态列表（房间号、房间码、当前人数、创建时间，展示最近 5 条活跃房间）
- 热门歌曲排行（排名、封面、歌曲名、歌手、播放量，展示 Top 5）

**业务规则**：
- 统计卡片数据实时从数据库聚合查询
- "在线"定义：用户在某虚拟房间内且未退出
- 排行榜按累计播放量降序排列
- 后续按需增加更多统计维度

**优先级**：高

---

#### 3.1.2 房间监控

**功能描述**：监控线上虚拟房间的实时状态，审批用户的开房申请，管理房间生命周期。

**子功能 3.1.2.1 活跃房间列表**

**输入**：
- 筛选条件：状态（全部 / 进行中 / 空闲待关闭）
- 搜索条件：房间码关键字

**输出**：
- 房间列表表格：房间码、当前人数、房间状态、创建时间、操作按钮
- 状态标签：进行中（绿色）、空闲待关闭（金色，显示倒计时）
- 操作按钮：关闭房间

**业务规则**：
- 房间码为 6 位随机数字/字母组合，房间创建后生成，关闭前保持不变
- "空闲待关闭"表示所有用户已退出，显示 5 分钟倒计时
- 倒计时期间如有用户加入，计时器重置，状态变回"进行中"
- 管理员可手动强制关闭房间，关闭后房间码作废
- 关闭的房间不在列表中展示

---

**子功能 3.1.2.2 开房申请审批**

**输入**：
- 无（自动加载待审批列表）

**输出**：
- 待审批列表：申请用户、申请时间、操作按钮（通过 / 拒绝）
- 无待审批时显示"暂无申请"

**业务规则**：
- 用户在 Web 端提交开房申请后，待审批列表实时更新
- 管理员点击"通过"：系统分配一个空房间，生成 6 位随机房间码，房间状态变为"进行中"
- 管理员点击"拒绝"：申请标记为已拒绝，用户端收到提示
- 管理员也可在活跃房间列表中主动开启房间（不需用户申请）

**优先级**：高

---

#### 3.1.3 歌曲管理

**功能描述**：维护歌曲库的增删改查，包含歌曲列表、新增歌曲弹窗、歌曲详情页三个层级。

---

**3.1.3.1 歌曲列表**

**输入**：
- 搜索：按歌曲名或歌手名模糊搜索
- 筛选：按流派分类筛选

**输出**：
- 3 张统计卡片：总曲目数、本周新增数、今日播放量
- 歌曲列表表格：封面缩略图、歌曲名、歌手、流派、状态、操作按钮（详情 / 删除）
- 支持分页浏览

**业务规则**：
- 歌曲状态分为：已上架（可被点歌）、已下架（不可见）
- 删除为软删除（状态标记为 disabled），不物理删除数据
- 流派预设值：流行、摇滚、民谣、电子、R&B、嘻哈、古典
- 时长以"分:秒"格式展示，存储以秒为单位
- 操作列："详情"按钮跳转到歌曲详情页，"删除"按钮触发确认弹窗后软删除

---

**3.1.3.2 新增歌曲弹窗**

**触发方式**：歌曲列表页点击"新增歌曲"按钮

**输入**：
- 歌曲名称（文本，必填）
- 歌手（文本，必填）
- 风格（下拉选择，必填，预设值同流派列表）
- 语种（下拉选择，必填，预设值：中文、英文、日文、韩文、其他）
- 音乐文件（文件上传，必填）
  - 接受格式：.mp3
  - 最大大小：30MB
  - 上传接口：`POST /api/upload/music`
  - 上传后返回 `url`，存入 `MediaUrl` 字段
- 封面图片（文件上传，可选）
  - 接受格式：.jpg, .jpeg, .png, .webp
  - 最大大小：5MB
  - 上传接口：`POST /api/upload/cover`
  - 未上传时使用默认封面 `/uploads/covers/default.jpg`
  - 文件丢失回退规则见 3.3 图片资源回退规则

**自动填充**：
- 时长（秒）：从上传的 MP3 文件自动读取，填入 `Duration` 字段
- 文件大小：从上传文件自动获取，填入 `FileSize` 字段
- 歌名和歌手：按以下优先级自动识别（仅新增模式，且对应字段为空时触发）
  1. **ID3 标签**：读取 MP3 文件内嵌的 `title` 和 `artist` 字段
  2. **文件名拆解**：ID3 标签缺失时，按 ` - `（减号两侧空格）拆分文件名，前半为歌手、后半为歌名；同时去除方括号标签（如 `[mqms]`）和扩展名
  3. 兜底：歌名取清理后的文件名，歌手留空
  - 详情页替换音乐文件时不触发此逻辑
- 封面图：从 MP3 文件 ID3 标签内嵌图片自动读取（仅新增模式，且封面为空时触发）
  - 读取成功后自动上传到服务器，填入封面字段
  - 无内嵌图片则跳过，不报错

**输出**：
- 上传过程中显示上传进度
- 上传成功后封面显示缩略图预览
- 保存成功后关闭弹窗，刷新歌曲列表

**业务规则**：
- 音乐文件为必填项，封面为可选项（未设置时使用默认封面 default.jpg）
- 文件上传与歌曲信息保存为两步：先上传文件获取 URL，再提交歌曲表单
- 文件上传失败时不提交表单，显示错误提示
- 时长从 MP3 文件 ID3 标签或文件头自动解析，无需用户手动输入
- 保存成功后关闭弹窗，刷新歌曲列表

---

**3.1.3.3 歌曲详情页**

**触发方式**：歌曲列表页点击某行的"详情"按钮

**展示字段（只读模式）**：

| 字段 | 来源 | 说明 |
|------|------|------|
| 封面 | `CoverUrl` | 大尺寸展示，无封面或文件丢失按 3.3 规则回退 |
| 歌曲名称 | `Title` | — |
| 歌手 | `Artist` | — |
| 文件大小 | `FileSize` | 上传时自动记录，格式化显示（如 4.2 MB） |
| 时长 | `Duration` | 格式化为"分:秒" |
| 风格 | `Genre` | — |
| 语种 | `Language` | — |
| 播放量 | `PlayCount` | 只读，实时数据 |
| 收藏数 | `Favorites` 表 COUNT | 实时查询，不存储冗余字段 |
| 评分 | 占位数据 | 展示字段，当前硬编码为 0 |
| 评论数 | 占位数据 | 展示字段，当前硬编码为 0 |
| 榜单排行 | 根据 `PlayCount` 实时计算 | 按播放量降序排名，显示当前排名 |
| 状态 | `Status` | 已上架 / 已下架 |
| 创建时间 | `CreatedAt` | 格式化展示 |

**修改模式**：
- 点击"修改"按钮进入编辑模式，"修改"按钮变为"保存"和"取消"
- 可修改字段：
  - 封面：可重新上传替换，上传接口同新增流程
  - 歌曲名称、歌手、风格、语种：文本/下拉直接修改
- 音乐文件替换：
  - 允许替换（支持换同首歌不同品质的场景）
  - 点击"替换音乐"按钮触发文件选择
  - 选择后弹出确认弹窗："替换后原文件将被删除，确定继续？"
  - 确认后上传新文件，删除旧文件，时长和文件大小自动更新
- 不可修改字段：播放量、收藏数、评分、评论数、榜单排行、创建时间

**业务规则**：
- 详情页为独立页面（非弹窗），从歌曲列表点击进入
- 详情页提供"返回列表"按钮
- 评分和评论数当前为占位展示（值为 0），未来有完整功能时再对接真实数据
- 收藏数通过 `SELECT COUNT(*) FROM Favorites WHERE SongId = @id` 实时查询
- 榜单排行通过 `SELECT COUNT(*) + 1 FROM Songs WHERE PlayCount > @playCount AND Status = 'active'` 实时计算
- 替换音乐文件时，先上传新文件成功后再删除旧文件，避免数据丢失
- 封面和音乐文件的旧文件在替换后从服务器物理删除（`wwwroot/uploads/` 下对应的文件）

---

**优先级**：高

---

#### 3.1.4 反馈管理

**功能描述**：管理用户提交的歌曲添加反馈，管理员审批后可将歌曲加入曲库。

**输入**：
- 筛选条件：状态（全部 / 待处理 / 已处理）
- 搜索：按歌曲名或用户名搜索

**输出**：
- 反馈列表表格：反馈用户、歌曲名、歌手（如有）、反馈类型、提交时间、状态、操作按钮
- 操作按钮：标记为已处理

**反馈类型**（预设）：
- 请求添加歌曲（主要类型）
- 歌曲信息纠错
- 其他建议

**业务规则**：
- 反馈来源：Web 端用户通过反馈按钮提交
- 反馈状态：待处理 / 已处理
- 管理员标记"已处理"后，反馈状态变为已处理，不再出现在待处理列表中
- 管理员处理反馈后，可手动前往歌曲管理添加对应歌曲（反馈处理与歌曲添加为两步操作）
- 反馈管理也可通过右上角通知图标快速访问：点击通知图标弹出待处理反馈列表，可直接标记已处理
- 通知图标显示待处理反馈数量角标

**优先级**：中

---

#### 3.1.5 账户管理

**功能描述**：管理用户账户信息和访问权限。

**输入**：
- 搜索：按用户名搜索
- 筛选：状态（全部 / 启用 / 禁用）

**输出**：
- 用户列表表格：用户名（含头像）、注册方式、状态、创建时间、操作按钮
- 已禁用用户行降低透明度

**业务规则**：
- 用户状态：启用 / 禁用
- 支持启用/禁用用户账户切换，点击禁用时弹出确认提示
- 如果该用户当前在某个房间中，确认弹窗显示"该用户当前在房间中，禁用后将退出房间。确认禁用？"
- 禁用后，该用户自动退出当前所在房间
- 被禁用用户无法登录 Web 端
- 用户注册方式包括：手机号、邮箱、用户名+密码

**优先级**：高

---

#### 3.1.6 系统设置

**功能描述**：配置平台的基础运营参数、操作日志查看及敏感操作验证策略。

**输入/输出**（四个配置区块）：

**区块 1 - 管理员账号**：
- 管理员用户名（文本，修改需密码确认）
- 修改管理员密码（输入当前密码 + 新密码 + 确认密码）

**区块 2 - 基本信息**：
- 平台名称（文本）
- 联系方式（文本）

**区块 3 - 安全设置**：
- 日志保留期限（下拉选择：30天 / 90天 / 180天 / 永久）
- 敏感操作二次验证（勾选制，可按操作类型独立配置）
  - 可配置项：禁用/启用用户、关闭房间、修改系统设置、修改管理员账号
  - 每项为独立复选框，默认全部勾选
  - 开启后对应操作需输入当前管理员密码二次确认

**区块 4 - 操作日志**：
- 日志列表（分页展示，所有管理员可查看）
  - 展示字段：时间、操作人、操作类型、操作对象、详情
  - 筛选条件：操作类型（下拉选择）、日期范围、操作人（下拉选择）
  - 记录的操作类型：禁用/启用用户、上下架歌曲、关闭房间、审批开房申请、处理反馈、修改系统设置、登录/登出
- 日志保留规则：按区块 3 中设置的日志保留期限自动清理过期日志

**业务规则**：
- 管理员账号在系统设置中单独管理，不在账户管理列表中展示
- 管理员账号不可禁用，禁用操作仅对普通用户生效
- 修改管理员用户名或密码需密码验证（输入当前密码）
- 操作日志为只读展示，不可修改或删除
- 操作日志在每次成功执行操作后自动写入，异步写入不影响主流程性能
- 敏感操作二次验证的配置修改本身也属于敏感操作（防止恶意关闭验证）
- 所有设置以键值对形式存储在数据库中
- 修改后点击"保存设置"按钮生效
- 保存成功后显示确认提示

**优先级**：中

### 3.2 Web 端功能

#### 3.2.1 注册

**功能描述**：新用户创建账户的入口。

**触发方式**：登录页点击"注册"链接

**输入**：
- 注册方式（三选一）：
  - 用户名 + 密码
  - 手机号 + 密码
  - 邮箱 + 密码
- 昵称（文本，必填）

**输出**：
- 注册成功：自动跳转到登录页
- 注册失败：显示错误提示（用户名/手机号/邮箱已存在等）

**业务规则**：
- 不需要手机验证码或邮箱验证码
- 注册方式只是标识，不影响功能权限
- 根据注册方式决定必填项（详见 9.2 注册逻辑）
- 用户名、手机号、邮箱有值时各自不可重复
- 注册成功后用户状态默认为"启用"

**优先级**：高

---

#### 3.2.2 登录

**功能描述**：用户进入系统的入口页面。

**输入**：
- 账号（用户名 / 手机号 / 邮箱）
- 密码

**输出**：
- 登录成功：弹出房间选择窗口
- 登录失败：显示错误提示

**房间选择窗口**（登录成功后弹出）：
- 两个选项：
  1. **输入房间码**：输入 6 位房间码，点击"加入"
  2. **申请开房间**：点击后提交开房申请，等待管理员审批，显示"申请已提交，等待管理员审批"
- 用户也可选择稍后再进入（关闭弹窗，进入探索歌曲页面）

**业务规则**：
- 登录成功后，服务端返回 JWT Token，前端存储到 localStorage
- Token 过期后自动跳转回登录页
- 登录页不显示侧边栏和底部播放器
- 输入房间码加入时，校验房间码是否存在且状态为"进行中"
- 用户同一时间只能在一个房间内，若已在某房间中再次加入，自动退出原房间
- 申请开房间后，管理员在房间监控页的待审批列表中可看到该申请

**优先级**：高

---

#### 3.2.3 探索歌曲

**功能描述**：用户浏览和搜索歌曲库的核心页面。

**输入**：
- 搜索：按歌曲名或歌手名搜索
- 流派筛选：点击流派标签筛选（全部、流行、摇滚、民谣等）

**输出**：
- 流派筛选标签行（横向滚动的药丸按钮）
- 歌曲列表：封面占位图、歌曲名、歌手、时长、收藏按钮、"点歌"按钮
- 页面右下角："反馈"浮动按钮

**业务规则**：

#### 3.2.7 底部播放器栏（全局组件）

**功能描述**：所有 Web 端页面（除登录/注册页）底部固定的悬浮胶囊播放器，基于 HTML5 `<audio>` 实现真实音频播放，切页面不中断。点击封面可展开为全屏歌词页。

**数据来源**：
- 播放队列：当前房间的已点歌曲列表
- 音频 URL：歌曲的 `MediaUrl` 字段，指向 `wwwroot/uploads/music/` 下已上传的 MP3 文件

**胶囊播放器（默认状态）**：
- 底部居中浮动药丸形容器（`max-width: 560px`，`bottom: 24px`）
- 左侧：圆形封面（`56×56px`，圆角 `rounded-full`），悬浮时放大 + 显示上箭头图标
- 中间：歌名 + 歌手名（单行省略）+ 内嵌进度条（1px 渐变色，悬浮显示拖拽手柄）
- 右侧：三键控制（上一首 / 播放暂停 / 下一首）+ 时间标签
- 毛玻璃背景（`bg-white/80 backdrop-blur-2xl`）

**空队列状态**：
- 胶囊播放器始终显示，不隐藏
- 歌曲信息区显示占位文字"未在播放"
- 封面不可点击（无 `cursor-pointer`），播放按钮置灰
- 进度条置灰

**歌词页（点击封面滑入）**：
- 触发条件：有歌曲播放时，点击胶囊播放器的圆形封面
- 全屏覆盖层（`z-index: 60`），从底部滑入（`translateY` 动画，0.4s 缓动）
- 背景：当前歌曲封面模糊放大（`blur-3xl`）+ 半透明黑色遮罩 + `backdrop-blur-xl` 毛玻璃
- 顶部：收起按钮（下箭头，点击关闭歌词页）+ "正在播放"标签 + 歌手名
- 中间：大封面（`256×256px`，`rounded-2xl`，阴影）+ 歌名 + 歌手名 + 歌词占位区域
- 底部：粗进度条（1.5px，悬浮显示拖拽手柄）+ 时间标签 + 实心圆播放按钮（白色，`56×56px`）+ 上/下一首按钮
- 关闭方式：点击顶部收起按钮 / 按 Escape 键
- 封面点击仅在 `player.hasTrack` 为 true 时生效，无歌曲时不响应

**播放行为**：
- 点击"点歌"：歌曲加入播放队列；若队列此前为空，自动播放该歌曲
- 播放队列中有多首歌曲时，当前歌曲播放完毕自动播放下一首
- 上一首/下一首：切换到队列中上一首/下一首歌曲；已在队列首尾时按钮置灰
- 播放/暂停：点击切换播放状态，图标同步切换
- 进度条（胶囊和歌词页共用）：
  - 实时显示当前播放进度
  - 支持拖拽定位到指定时间点
- 音量控制已从播放器栏移除（当前版本精简）

**业务规则**：
- 播放器使用全局单例的 `<audio>` 实例，跨页面共享，切页面不中断播放
- 播放状态存储在 Pinia store 中，页面切换后恢复
- 音频加载失败时，自动跳到下一首
- 队列中所有歌曲播放完毕后，播放器回到空队列状态
- 胶囊播放器和歌词页共享同一个 Pinia player store，进度实时同步

**优先级**：高

#### 3.2.8 侧边导航栏（全局组件）

**功能描述**：Web 端所有页面（除登录/注册页）左侧固定显示的导航栏，支持收起/展开切换。

### 3.3 图片资源展示与回退规则（通用）

**适用范围**：所有展示头像或封面图片的位置，包括管理端账户列表、编辑弹窗、歌曲列表、歌曲详情页、Web 端歌曲卡片。

**封面展示形状**：
- 所有位置的封面图片统一以 **1:1 正方形**展示
- 使用 `object-cover` 裁剪填充，圆角根据容器定义
- 不得使用圆形、圆角矩形等非正方形形状展示封面

**头像展示形状**：
- 所有位置的头像图片统一以 **圆形**展示

**文件回退规则**：

| 场景 | 行为 |
|------|------|
| 未上传自定义图片 | 显示默认图（头像：`/uploads/avatars/default.jpg`，封面：`/uploads/covers/default.jpg`） |
| 已上传但文件被物理删除（404） | 回退显示对应的默认图 |
| 默认图也被删除（404） | 浏览器显示裂图（原生 broken image 表现） |

**前端实现要求**：
- `<img>` 标签的 `src` 指向数据库中存储的 URL（自定义路径或默认路径）
- 绑定 `@error` 事件：当图片加载失败时，将 `src` 切换为默认图 URL
- 如果默认图也加载失败，不做额外处理，由浏览器显示原生裂图
- **不得**回退到首字母、音符图标等非图片元素

**优先级**：高

## 6. 非功能性需求

### 6.1 性能
- 页面首次加载时间 < 3 秒（本地开发环境）
- API 接口响应时间 < 500ms（Mock 数据模式）
- 列表分页加载，单页不超过 10 条

### 6.2 兼容性
- 支持 Chrome 最新版本（主要开发和测试浏览器）
- 支持 Edge 最新版本

### 6.3 安全性
- 密码不得明文存储，使用哈希算法
- API 接口使用 JWT Token 认证
- 敏感操作（禁用用户、关闭房间、修改设置）需要确认弹窗
- 管理员敏感操作需输入当前管理员密码二次确认（可在系统设置中配置）

### 6.4 可用性
- 所有交互操作提供视觉反馈（按钮按下缩放 98%）
- 加载中状态显示骨架屏或加载指示器
- 操作结果使用消息提示（成功/失败）

### 6.5 代码质量
- 前端使用 TypeScript 严格模式
- 后端使用 C# 强类型
- API 接口统一错误格式返回

### 6.6 视觉设计

**全局背景**：Web 端使用多层浮动色块动画背景，替代简单渐变。5 个不同大小的半透明圆形色块（`#71fcfe` 及其变体），通过 CSS `filter: blur()` 产生柔和光晕效果，各自独立动画驱动（`@keyframes` + `translate` + `scale`），营造景深感：

| 色块 | 尺寸 | 动画周期 | 活动范围 | 景深定位 |
|------|------|----------|----------|----------|
| 大色块 ×2 | 500-600px | 15s | ±15vw / ±12vh | 远景，稳重缓慢 |
| 中色块 ×2 | 400-450px | 10s | ±25vw / ±20vh | 中景，适中速度 |
| 小色块 ×1 | 350px | 8s | ±40vw / ±30vh | 近景，快速，可跑出屏幕 |

色块容器固定定位（`position: fixed`），`z-index: 0`，`pointer-events: none` 不阻挡交互。

**毛玻璃效果**：以下组件使用 `background: rgba(255,255,255,0.7); backdrop-filter: blur(20px)` 实现毛玻璃，让底层色块透出：
- 顶部导航栏
- 侧边导航栏
- 底部播放器栏
- 登录/注册页卡片
- 当前房间页：正在播放卡片、聊天面板、房间信息标签
- 流派筛选标签行、榜单周期切换容器

**歌曲行悬浮预览**：探索歌曲、热门榜单、我的收藏、当前房间播放列表中的歌曲条目，悬浮时单行独立触发预览效果：`hover:glass hover:shadow-lg hover:scale-[1.02]`，仅当前行变化，不影响其他行。不再使用封面悬浮放大效果。

**侧边栏收起**：侧边导航栏支持收起/展开切换。收起后宽度从 256px（`w-64`）变为 80px（`w-20`），仅显示图标。切换按钮固定在侧边栏右边缘、垂直居中位置，点击触发收起/展开，箭头图标随状态旋转 180°。主内容区同步调整左边距（`ml-64` ↔ `ml-20`），带 300ms 过渡动画。

**登录/注册页**：移除原有装饰性模糊圆形，统一由全局色块背景覆盖，卡片使用毛玻璃效果。

### 6.7 响应式布局
- 管理端和 Web 端页面根据窗口大小自动调整布局
- 内容区最小尺寸：1024×600px
- 窗口缩小至最小尺寸时，内容区停止缩放，出现滚动条供用户拖动查看
- 全局生效，不限于特定页面

## 10. SignalR Hub 设计

### 10.1 概要

使用**单个 Hub** 处理房间生命周期事件，连接路径 `/hubs/ktv`。

用户加入房间时自动加入 SignalR Group `room-{roomId}`，退出时离开。

**注意**：实时聊天已改为 REST API + 轮询方案（每 2 秒），不通过 SignalR 传输。

### 10.2 客户端 → 服务端方法

| 方法 | 参数 | 说明 |
|------|------|------|
| `JoinRoom` | `roomId: int` | 加入房间组，广播 UserJoined |
| `LeaveRoom` | `roomId: int` | 退出房间组，广播 UserLeft |

### 10.3 服务端 → 客户端方法

| 方法 | 参数 | 触发时机 |
|------|------|----------|
| `UserJoined` | `nickname, userCount` | 有人加入房间 |
| `UserLeft` | `nickname, userCount` | 有人退出房间 |
| `QueueUpdated` | `queue: PlayQueueItem[]` | 点歌/删除/置顶后队列变化 |
| `RoomClosed` | 无 | 管理员关闭房间或倒计时结束 |
| `RoomApproved` | `roomCode: string` | 管理员审批通过（推送给申请者） |

### 10.4 聊天 REST API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/chat/send` | POST | 发送消息（body: roomId, message） |
| `/api/chat/messages?roomId=X` | GET | 获取房间最近 50 条消息 |

- 消息持久化到 ChatMessages 表
- 消息长度限制 200 字符
- 服务端做敏感词过滤（命中则替换为 `*`）
