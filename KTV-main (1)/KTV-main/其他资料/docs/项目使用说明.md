# 声域友 — 线上社交 KTV 平台 使用说明

> **面向读者**：组员、课程老师、答辩评委
> **最后更新**：2026-05-24
> **项目版本**：v2.1.0

---

## 1. 项目简介

声域友是一套**线上社交 KTV 平台**。用户通过浏览器创建或加入虚拟房间，在房间内与朋友一起点歌、排队、实时聊天，享受线上 KTV 体验。

系统分为两个端：

| 端 | 地址 | 使用者 |
|----|------|--------|
| **Web 端** | `http://localhost:5174` | 普通用户（唱歌的人） |
| **管理后台** | `http://localhost:5173` | 管理员（运营平台的人） |
| **后端 API** | `http://localhost:5276` | 两个前端共同依赖 |

## 2. 部署指南（从零到运行）

### 2.1 前置条件

| 依赖 | 最低版本 | 用途 |
|------|---------|------|
| [.NET 8.0 SDK](https://dotnet.microsoft.com/download/dotnet/8.0) | 8.0 | 运行后端 |
| [Node.js](https://nodejs.org/) | 20.19+ 或 22.12+ | 运行前端（Vite 8 要求） |
| SQL Server | 2019+ | 数据库（见下方配置说明） |

### 2.2 数据库配置

连接字符串在 [backend/appsettings.json](../backend/appsettings.json) 中，根据你的 SQL Server 环境修改 `DefaultConnection` 的值：

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost\\SQLEXPRESS;Database=KTVSystem;Trusted_Connection=true;TrustServerCertificate=true;"
  }
}
```

**不同环境的连接字符串写法：**

| 场景 | Server 值 |
|------|-----------|
| SQL Server 默认实例 | `Server=localhost` |
| SQL Server Express 命名实例 | `Server=localhost\\SQLEXPRESS` |
| SQL Server Developer 命名实例 | `Server=localhost\\MSSQLSERVER01` |
| 远程服务器 | `Server=192.168.1.100,1433` |
| SQL Server + 用户名密码登录 | `Server=localhost;Database=KTVSystem;User Id=sa;Password=你的密码;TrustServerCertificate=true;` |

> **Windows 身份验证**（默认）：使用 `Trusted_Connection=true`，当前 Windows 登录账号需有 SQL Server 权限。
> **SQL Server 身份验证**：去掉 `Trusted_Connection=true`，改为 `User Id=sa;Password=xxx`。

### 2.3 初始化数据库（首次部署必做）

**方式一：手动执行 SQL 脚本（推荐）**

1. 打开 SQL Server Management Studio (SSMS) 或 Azure Data Studio
2. 连接到你的 SQL Server 实例
3. 打开 [database/init.sql](../database/init.sql)，点击执行
4. 看到 `数据库初始化完成!` 即成功

也可用命令行：

```bash
sqlcmd -S localhost\SQLEXPRESS -i database\init.sql
```

**方式二：让后端自动建表**

后端首次启动时会自动检测并创建缺失的表和字段。但种子数据（管理员账号、系统设置）需要手动插入，推荐使用方式一。

### 2.4 一键启动

**Windows 用户** — 在项目根目录创建 `start.bat`：

```bat
@echo off
echo [1/3] 启动后端...
start "Backend" cmd /k "cd backend && dotnet run"
timeout /t 5
echo [2/3] 启动 Web 端...
start "Web" cmd /k "cd web && npm install && npm run dev"
echo [3/3] 启动管理后台...
start "Admin" cmd /k "cd admin && npm install && npm run dev"
echo.
echo 全部启动完成！
echo   后端 API:   http://localhost:5276
echo   Web 端:     http://localhost:5174
echo   管理后台:   http://localhost:5173
pause
```

**手动启动（三个终端）：**

```bash
# 终端 1 — 启动后端
cd backend
dotnet run
# 看到 "Now listening on: http://localhost:5276" 即成功

# 终端 2 — 启动 Web 端
cd web
npm install    # 首次需要
npm run dev
# 默认 http://localhost:5174

# 终端 3 — 启动管理后台
cd admin
npm install    # 首次需要
npm run dev
# 默认 http://localhost:5173
```

普通用户需在 Web 端自行注册。

### 2.6 前端环境变量

## 3. 管理后台使用说明

### 3.1 登录

打开 `http://localhost:5173`，输入管理员账号密码登录。

### 3.2 仪表盘

登录后默认进入仪表盘，展示四个核心数据：

| 卡片 | 含义 |
|------|------|
| 在线房间数 | 当前状态为"进行中"的虚拟房间数 |
| 在线用户数 | 当前在某个房间内的用户数 |
| 今日创建房间 | 今天新创建的房间总数 |
| 总注册用户 | 平台累计注册用户数 |

下方还有**最新房间动态**表格和**热门歌曲排行榜**（Top 5）。

所有数据每 3 秒自动刷新。

### 3.3 房间监控

管理线上虚拟房间的生命周期。

**活跃房间** 标签页：
- 查看所有房间：房间码、当前人数、状态、创建时间
- 支持按状态筛选：全部 / 进行中 / 空闲待关闭 / 已关闭
- 支持按房间码搜索
- **查看用户**：点击后弹窗显示该房间内所有用户
- **关闭房间**：强制关闭，房间内所有用户被踢出

**待审批** 标签页：
- 显示用户提交的开房申请
- **通过**：系统分配一个 6 位房间码，用户可凭码加入
- **拒绝**：申请作废
- 右上角角标显示待审批数量

### 3.4 歌曲管理

维护平台歌曲库。

**歌曲列表**：
- 3 张统计卡片：总曲目 / 本周新增 / 今日播放量
- 支持按歌名/歌手搜索
- 支持按状态筛选：全部 / 已上架 / 已下架
- 每首歌显示封面、歌名、歌手、流派、状态
- **详情**：进入歌曲详情页（可修改歌曲信息、替换封面/音乐文件）
- **删除**：软删除，歌曲状态变为 disabled

**新增歌曲**：
1. 点击"新增歌曲"按钮
2. 填写歌曲名称、歌手、风格、语种
3. 上传音乐文件（MP3，最大 30MB）— 必填
4. 上传封面图片（可选，支持 JPG/PNG/WebP）
5. 上传歌词文件（可选，`.lrc` 格式）
6. 系统自动从 MP3/FLAC 文件读取时长和封面（FLAC 内置封面会自动显示）
7. 点击保存

**歌曲详情页**：
- 查看完整歌曲信息（封面、歌名、歌手、时长、播放量、收藏数、排名等）
- 点击"修改"进入编辑模式，可修改歌名、歌手、风格、语种
- 可替换封面图片、音乐文件、歌词文件
- 替换音乐文件时会先确认，原文件自动删除

### 3.5 账户管理

管理普通用户账户。

- 用户列表：用户名（含头像）、状态（启用/禁用）、在线状态、所在房间、创建时间
- 支持按用户名搜索、按状态筛选
- **新增账户**：手动创建用户（含头像上传，支持裁剪）
- **编辑**：修改显示名称、手机号、头像
- **更改密码**：管理员可重置任意用户密码
- **禁用/启用**：切换用户状态。禁用后用户无法登录，已在房间中会被踢出

在线状态说明：
- 用户登录后 10 分钟内显示"在线"（绿色）
- 用户退出登录后立即显示"离线"
- 在线用户如果在某个房间中，"所在房间"列显示房间码

### 3.6 反馈管理

处理用户提交的歌曲相关反馈。

- 支持按状态筛选：全部 / 待处理 / 已处理
- 支持按歌曲名/用户名搜索
- 反馈类型：请求添加歌曲 / 歌曲信息纠错 / 其他建议
- **标记已处理**：点击后反馈状态变为"已处理"
- 处理反馈后，管理员可手动去歌曲管理中添加对应歌曲

### 3.7 系统设置

**管理员账号**：
- 修改管理员用户名（需输入当前密码确认）
- 修改管理员密码（需输入当前密码）

**基本信息**：
- 平台名称：显示在 Web 端顶部导航栏
- 联系方式

**安全设置**：
- 操作日志保留期限：30天 / 90天 / 180天 / 永久
- 敏感操作二次验证：勾选后执行对应操作需输入管理员密码
  - 可配置项：禁用/启用用户、关闭房间、修改设置、修改管理员账号

**操作日志**：
- 记录所有管理员操作（登录、禁用用户、关闭房间、审批、处理反馈等）
- 支持按操作类型、操作人、日期范围筛选
- 只读，不可修改或删除

## 4. Web 端使用说明

### 4.1 注册

打开 `http://localhost:5174`，点击登录页下方的"注册"链接。

支持三种注册方式（任选其一）：
- 用户名 + 密码
- 手机号 + 密码
- 邮箱 + 密码

昵称为必填项。注册成功后跳转到登录页。

### 4.2 登录与加入房间

1. 输入账号（用户名/手机号/邮箱）和密码
2. 登录成功后弹出**房间选择窗口**：
   - **输入房间码**：输入 6 位房间码加入已有房间
   - **申请开房间**：提交申请，等待管理员审批
   - **稍后进入**：关闭弹窗，先去浏览歌曲

### 4.3 探索歌曲

平台核心页面，浏览和搜索歌曲库。

- 顶部：流派筛选标签（全部 / 流行 / 摇滚 / 民谣 / ...）
- 歌曲列表：封面、歌名、歌手、时长
- **收藏按钮**（心形图标）：点击切换收藏状态，红色实心 = 已收藏
- **点歌按钮**：将歌曲加入当前房间播放队列

**反馈浮动按钮**（右下角）：
- 点击弹出反馈弹窗
- 选择反馈类型（请求添加歌曲 / 歌曲信息纠错 / 其他建议）
- 填写歌曲名、歌手、补充说明
- 提交后管理员在管理后台可看到

### 4.4 当前房间

进入房间后的核心页面，三个区域：

**房间信息栏**：
- 房间码（6 位）
- 在线人数（实时更新）
- 已点歌曲数量
- 退出房间按钮

**播放队列**：
- 正在播放卡片：大封面、歌名、歌手、进度条、切歌按钮
- 队列列表：序号、封面、歌名、歌手、点播者
- 点击歌曲可切换播放
- 悬浮显示删除按钮

**实时聊天**（右侧）：
- 消息列表：昵称 + 内容 + 时间
- 系统消息居中显示（如"XXX 加入了房间"）
- 底部输入框，点击发送或回车
- 消息每 2 秒自动刷新

### 4.5 热门榜单

展示最热门的歌曲排行，支持切换日榜 / 周榜。

- Top 10 排行，显示排名、封面、歌名、歌手、播放量
- 第 1 名金色排名标识
- 每首歌都有"点歌"按钮

### 4.6 我的收藏

展示当前用户收藏的所有歌曲。

- 页面标题显示收藏数量（如"我的收藏 · 12 首"）
- 悬浮显示取消收藏按钮和点歌按钮
- 取消收藏后歌曲从列表移除

### 4.7 底部播放器栏

所有页面底部固定显示的悬浮胶囊播放器，切页面不中断播放。

- 圆形封面（56px）+ 歌名/歌手 + 内嵌进度条 + 上一首/播放暂停/下一首 + 时间
- 点击封面 → 打开全屏歌词页

### 4.8 全屏歌词页

点击胶囊封面从底部滑入，封面模糊放大做背景 + 毛玻璃遮罩。

**左侧面板（38%）**：
- 大封面（播放时圆角方形，暂停时圆形）
- 歌名/歌手
- 进度条（可拖拽）
- 播放控制：上一首 / 播放暂停 / 下一首
- 播放模式切换：顺序播放 → 列表循环 → 单曲循环 → 随机播放
- 音量控制（悬浮展开滑块）

**右侧面板**：
- 漏斗式歌词：当前行放大加亮，远处缩放渐隐，点击歌词跳转到对应时间
- 播放列表侧栏（288px）：拖拽排序、收藏、自动滚屏到当前歌曲

关闭方式：顶部收起按钮 / Escape 键。

### 4.9 播放列表

播放列表分两个区域：
- **正在播放**：固定在顶部的卡片，显示当前播放歌曲，不可拖动
- **接下来**：排队歌曲列表，可拖拽排序，可收藏，点击歌曲立即移到正在播放位置

拖拽规则：
- 当前播放歌曲不可拖动
- 其他歌曲不可拖到正在播放位置
- 拖拽排序结果同步到服务器，其他用户可见

### 4.10 退出登录

点击右上角头像 → 下拉菜单 → "退出登录"。

### 4.8 退出登录

点击右上角头像 → 下拉菜单 → "退出登录"。

退出时：
- 自动退出当前房间
- 通知后端清除在线状态
- 跳转到登录页

## 5. 实时同步机制

系统中所有"实时"体验都基于**轮询**实现：

| 数据 | 轮询间隔 | 技术方案 |
|------|---------|---------|
| 播放队列 | 2 秒 | `GET /api/room/queue` |
| 聊天消息 | 2 秒 | `GET /api/chat/messages` |
| 房间信息 | 2 秒 | `GET /api/room/current` |
| 管理后台 | 3 秒 | 各列表页 `setInterval` |

原理：用户 A 点歌 → 写入数据库 → 用户 B 的下次轮询查到新数据 → 更新 UI。

最大延迟 = 轮询间隔（2-3 秒）。

---

## 6. 关键业务规则

1. **一个用户同时只能在一个房间**：加入新房间自动退出旧房间
2. **同一首歌在同一个房间只能被点一次**（未播放状态）
3. **歌曲播放后从队列移除**，状态变为 removed
4. **用户退出房间后**，其点播的歌曲从队列中清除
5. **禁用用户**会自动踢出所在房间，且无法登录
6. **关闭房间**会踢出所有用户，房间码作废
7. **聊天消息**最长 200 字符，含敏感词自动过滤
