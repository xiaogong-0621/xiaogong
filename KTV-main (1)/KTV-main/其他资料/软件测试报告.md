# 声域友线上社交KTV平台 — 软件测试报告

## 1 引言

### 1.1 测试目的

本测试报告旨在对声域友线上社交KTV平台进行全面测试，验证系统功能是否满足需求规格说明书中的要求，确保系统在交付前达到预期的质量标准。

### 1.2 测试范围

测试范围涵盖以下核心功能模块：

| 模块 | 测试内容 |
|------|----------|
| 用户管理模块 | 注册、登录、信息修改、密码修改 |
| 歌曲管理模块 | 歌曲列表、搜索、上传、编辑、删除 |
| 房间管理模块 | 创建房间、加入房间、退出房间、关闭房间 |
| 播放控制模块 | 点歌、播放、暂停、切歌、播放模式切换 |
| 聊天模块 | 发送消息、接收消息、系统消息 |

## 3 测试策略

### 3.1 测试方法

本系统采用以下测试方法：

**（1）黑盒测试**：不考虑程序内部结构，仅根据输入输出验证系统功能是否符合需求规格说明书。

**（2）等价类划分**：将输入数据划分为有效等价类和无效等价类，每类选取代表值进行测试。

**（3）边界值分析**：对输入条件的边界值进行测试，检测边界处理是否正确。

**（4）性能测试**：对关键接口进行并发测试，验证系统的响应时间和吞吐量。

### 3.2 测试级别

| 测试级别 | 说明 |
|----------|------|
| 单元测试 | 对单个函数或方法进行测试 |
| 集成测试 | 测试模块间接口和交互 |
| 系统测试 | 对整个系统进行端到端测试 |
| 验收测试 | 模拟用户操作验证系统可用性 |

### 3.3 测试通过准则

1. 所有测试用例通过率达到100%
2. 无严重缺陷遗留
3. 核心业务功能全部可用
4. API接口响应时间小于500ms

### 5.2 性能分析

1. **歌曲列表接口（GET /api/Songs）**：在高并发场景下表现良好，平均响应时间85ms，满足性能要求。

2. **播放队列接口（GET /api/Room/queue）**：作为高频轮询接口，平均响应时间45ms，表现优秀。

3. **聊天发送接口（POST /api/Chat/send）**：平均响应时间38ms，延迟低，用户体验良好。

4. **登录接口（POST /api/Auth/login）**：包含JWT生成逻辑，平均响应时间120ms，在可接受范围内。

5. 所有接口成功率均为100%，未出现超时或连接拒绝的情况。
