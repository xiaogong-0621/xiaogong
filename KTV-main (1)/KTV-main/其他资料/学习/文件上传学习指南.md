# 文件上传完全指南（零基础版）

> 写给没有任何文件上传经验的开发者。
> 读完这篇你就能理解：文件怎么从浏览器跑到服务器、中间经过了什么、代码怎么写。

---

## 一、最基础的问题：文件上传到底是什么？

### 普通请求 vs 文件上传请求

你之前写的 API 请求，发的都是 **JSON**，长这样：

```
POST /api/accounts/1
Content-Type: application/json

{"displayName": "张三", "phone": "13800000001"}
```

JSON 本质就是一段**文本**，很小，几十 KB 就到头了。

但文件不一样 — 一张照片几 MB，一首歌十几 MB。你不可能把一首 MP3 转成 JSON 文本发过去（技术上可以，但会膨胀 33%，非常蠢）。

所以浏览器发明了一种专门传文件的方式，叫 **multipart/form-data**。

### multipart/form-data 是什么？

"multipart" = 多部分。一个请求里可以塞多个"部分"，每个部分可以是文本字段，也可以是文件。

```
POST /api/upload/avatar
Content-Type: multipart/form-data; boundary=----WebKitFormBoundaryABC123

------WebKitFormBoundaryABC123
Content-Disposition: form-data; name="file"; filename="我的头像.jpg"
Content-Type: image/jpeg

[这里是一大坨二进制数据，就是照片的实际内容]
------WebKitFormBoundaryABC123--
```

**你不需要手动写这个格式**，浏览器和 axios 帮你自动生成。你只需要知道：
- JSON 请求：发文本，用 `axios.post(url, { key: value })`
- 文件请求：发文件，用 `axios.post(url, formData)`，其中 formData 是 FormData 对象

## 二、前端怎么做文件上传？

### 第 1 步：让用户选文件

浏览器提供了一个原生的文件选择器：

```html
<input type="file" />
```

用户点击后弹出系统文件选择框。选完文件后，浏览器会在 JS 里给你一个 **File 对象**。

```js
// 监听 change 事件，用户选完文件就触发
input.addEventListener('change', (event) => {
  const file = event.target.files[0]  // 用户选的文件
  console.log(file.name)    // "我的头像.jpg"
  console.log(file.size)    // 204800 (字节)
  console.log(file.type)    // "image/jpeg"
})
```

File 对象就是文件的"句柄" — 你可以读它的名字、大小、类型，也能把它发给服务器。

### 第 2 步：把文件塞进 FormData

```js
const formData = new FormData()          // 创建一个空的 FormData
formData.append('file', file)            // 把文件塞进去，'file' 是字段名
```

**为什么叫 `append`？** 因为 FormData 就像一个袋子，你可以往里面塞东西：
- `formData.append('file', file)` — 塞一个文件
- `formData.append('username', '张三')` — 塞一段文本
- 一个 FormData 可以塞多个东西

**为什么字段名叫 `'file'`？** 这必须和后端的参数名一致。后端写的是 `UploadAvatar(IFormFile file)`，所以前端 append 的 key 也必须是 `'file'`。名字对不上，后端就收不到。

### 第 3 步：发送请求

```js
import axios from 'axios'

const response = await axios.post('http://localhost:5000/api/upload/avatar', formData)
// response.data = { url: "/uploads/avatars/a1b2c3d4.jpg", fileName: "a1b2c3d4.jpg" }
```

**注意**：这里没有手动设置 Content-Type。axios 看到你传的是 FormData，会自动：
1. 设置 `Content-Type: multipart/form-data`
2. 加上 `boundary` 分隔符
3. 把文件的二进制数据编码进去

你什么都不用管，传 FormData 就行。

### 第 4 步：拿到 URL，保存到数据库

```js
// 上传文件，拿到 URL
const { url } = response.data   // "/uploads/avatars/a1b2c3d4.jpg"

// 更新用户信息，把 URL 存到数据库
await axios.put('/api/accounts/1', {
  displayName: '张三',
  avatarUrl: url                  // ← 存进去了
})
```

### 第 5 步：显示图片

```html
<!-- 从数据库读出 avatarUrl，拼上后端地址，给 img 标签 -->
<img :src="'http://localhost:5000' + user.avatarUrl" />
```

浏览器看到这个 URL，会向后端发 GET 请求。后端的 `UseStaticFiles()` 中间件会直接把文件内容返回，浏览器渲染出图片。

## 三、后端怎么做文件接收？

### ASP.NET Core 的文件接收

```csharp
[HttpPost("avatar")]
public async Task<IActionResult> UploadAvatar(IFormFile file)
//                              ↑ 这个参数名 'file' 必须和前端 formData.append('file', ...) 对应
```

ASP.NET 看到请求的 Content-Type 是 `multipart/form-data`，会自动解析请求体，把文件部分提取成 `IFormFile` 对象。

### IFormFile 有什么属性？

```csharp
file.FileName    // "我的头像.jpg"  — 用户原始文件名
file.Length      // 204800         — 文件大小（字节）
file.ContentType // "image/jpeg"   — 文件 MIME 类型
file.OpenStream() // 获取文件内容的流
```

### 保存文件到磁盘

```csharp
// 1. 生成唯一文件名（避免重名覆盖）
var fileName = Guid.NewGuid() + ".jpg";

// 2. 拼出保存路径
var savePath = Path.Combine("wwwroot", "uploads", "avatars", fileName);

// 3. 把上传的文件内容写入磁盘
using (var stream = new FileStream(savePath, FileMode.Create))
{
    await file.CopyToAsync(stream);
}

// 4. 返回 URL 给前端
return Ok(new { url = $"/uploads/avatars/{fileName}" });
```

### 为什么用 GUID 重命名？

用户上传的文件名可能包含：
- 中文字符：`我的头像.jpg`
- 空格：`my photo.jpg`
- 特殊字符：`a&b=1.jpg`
- 同名文件：两个人都传了 `avatar.jpg`

如果直接用原始文件名保存：
- URL 里有中文/空格，浏览器可能编码出错
- 同名文件会互相覆盖
- 文件名里有 `../` 可能造成路径穿越安全漏洞

用 GUID 重命名后：`a3f1b2c4-5d6e-7f8a-9b0c-1d2e3f4a5b6c.jpg`
- 永远不会重复（宇宙毁灭都不会）
- 不含任何特殊字符
- 无法通过文件名猜测其他文件

**原始文件名存在数据库就行**，磁盘上永远用 GUID。

## 四、UseStaticFiles 是什么？

ASP.NET Core 有一个"静态文件中间件"。它的作用是：

> 如果请求的 URL 对应 `wwwroot` 目录下的一个文件，直接返回文件内容，不走任何 Controller。

```
请求: GET http://localhost:5000/uploads/avatars/a1b2c3.jpg
                                     ↓
                              UseStaticFiles 拦截
                                     ↓
                        wwwroot/uploads/avatars/a1b2c3.jpg 文件存在？
                              ↓ 是              ↓ 否
                         返回文件内容        继续往下走（404）
```

**所以文件上传的本质就是**：
1. 把文件保存到 `wwwroot` 下的某个目录
2. 把路径告诉前端
3. 前端用这个路径作为 `<img src>` 或 `<audio src>`
4. 浏览器请求这个路径时，UseStaticFiles 直接返回文件

## 五、完整代码走一遍

### 前端（Vue 3 + TypeScript）

```vue
<template>
  <!-- 头像预览 -->
  <img :src="avatarDisplayUrl" class="w-16 h-16 rounded-full" />

  <!-- 文件选择器，选完自动触发上传 -->
  <input type="file" accept="image/*" @change="handleFileChange" />

  <!-- 保存按钮 -->
  <button @click="saveProfile">保存</button>
</template>

<script setup lang="ts">
import { ref, computed } from 'vue'
import axios from 'axios'

const avatarUrl = ref<string | null>(null)     // 上传后得到的 URL
const selectedFile = ref<File | null>(null)     // 用户选的文件

// 显示用的 URL：有上传的就用上传的，没有就用默认图
const avatarDisplayUrl = computed(() => {
  if (avatarUrl.value) return `http://localhost:5000${avatarUrl.value}`
  return 'http://localhost:5000/uploads/avatars/default.jpg'
})

// 用户选了文件
async function handleFileChange(event: Event) {
  const input = event.target as HTMLInputElement
  if (!input.files?.length) return
  selectedFile.value = input.files[0]

  // 立即上传，拿到 URL
  const formData = new FormData()
  formData.append('file', selectedFile.value)

  const { data } = await axios.post('http://localhost:5000/api/upload/avatar', formData)
  avatarUrl.value = data.url   // 例如 "/uploads/avatars/a1b2c3.jpg"
}

// 点保存，把 URL 存到数据库
async function saveProfile() {
  await axios.put('http://localhost:5000/api/accounts/1', {
    avatarUrl: avatarUrl.value
  })
}
</script>
```

### 后端（我们已经写好的 UploadController）

```csharp
[HttpPost("avatar")]
public async Task<IActionResult> UploadAvatar(IFormFile file)
{
    // 1. 检查有没有文件
    if (file == null || file.Length == 0)
        return BadRequest(new { message = "未选择文件" });

    // 2. 检查扩展名
    var ext = Path.GetExtension(file.FileName).ToLower();
    var allowed = new[] { ".jpg", ".jpeg", ".png", ".gif", ".webp" };
    if (!allowed.Contains(ext))
        return BadRequest(new { message = "不支持的格式" });

    // 3. 检查大小
    if (file.Length > 2 * 1024 * 1024)
        return BadRequest(new { message = "文件太大，最大 2MB" });

    // 4. GUID 重命名 + 保存
    var fileName = Guid.NewGuid() + ext;
    var savePath = Path.Combine(_env.WebRootPath, "uploads", "avatars", fileName);
    using var stream = new FileStream(savePath, FileMode.Create);
    await file.CopyToAsync(stream);

    // 5. 返回 URL
    return Ok(new { url = $"/uploads/avatars/{fileName}" });
}
```

## 六、常见问题 Q&A

### Q: 为什么不直接用 JSON 传文件？
A: 文件是二进制数据，JSON 只能传文本。虽然可以 Base64 编码后塞进 JSON，但会膨胀 33%，一首 10MB 的歌变成 13MB，而且编码解码浪费 CPU。multipart/form-data 是浏览器专门为传文件设计的，效率最高。

### Q: Content-Type 什么时候该手动设置？
A: **几乎 never**。传 JSON 时 axios 自动设 `application/json`，传 FormData 时 axios 自动设 `multipart/form-data`。你手动设反而容易出错。

### Q: 前端的 `formData.append('file', file)` 中的 `'file'` 能改吗？
A: 能改，但必须和后端参数名同步改。后端写的是 `IFormFile file`，所以前端 append 的 key 必须是 `'file'`。如果后端改成 `IFormFile avatar`，前端就要改成 `formData.append('avatar', file)`。

### Q: 文件上传有大小限制吗？
A: 三层限制：
1. **后端 Kestrel**：默认 30MB，我们在 Program.cs 里配置了
2. **后端代码**：在 Controller 里按类型检查（头像 2MB、封面 5MB、音乐 30MB）
3. **前端**：可以在选文件时检查 `file.size`，超限就提示用户

### Q: 多个文件怎么上传？
A: 两种方式：
- **一次一个请求传多个文件**：`formData.append('files', file1)` 和 `formData.append('files', file2)`，后端参数改成 `IFormFile[] files`
- **一次一个请求传一个文件，发多次请求**：前端循环调用上传接口

对你的项目来说，一次传一个就够了。

### Q: 上传进度怎么显示？
A: axios 支持 `onUploadProgress` 回调：
```js
axios.post('/api/upload/music', formData, {
  onUploadProgress: (progressEvent) => {
    const percent = Math.round((progressEvent.loaded / progressEvent.total) * 100)
    console.log(`上传进度: ${percent}%`)
  }
})
```

### Q: 上传失败了怎么办？
A: 用 try-catch 包裹上传请求，失败时显示错误提示。后端已经返回了具体的错误信息（格式不对、文件太大等），前端直接展示 `error.response.data.message` 就行。

### Q: 文件上传需要登录吗？
A: 看后端配置。我们的 UploadController 加了 `[Authorize]`，需要 JWT Token 才能访问。axios 的拦截器应该已经配好了自动带 Token，所以前端不需要额外处理。

## 七、关键概念速查表

| 概念 | 一句话解释 |
|------|-----------|
| **File 对象** | 浏览器里代表用户选中的文件，有名字、大小、类型 |
| **FormData** | 一个"袋子"，用来装要上传的文件和文本字段 |
| **multipart/form-data** | 文件上传专用的请求格式，FormData 自动变成这个格式 |
| **IFormFile** | ASP.NET 后端接收上传文件的类型 |
| **IWebHostEnvironment.WebRootPath** | 后端 wwwroot 目录的磁盘绝对路径 |
| **UseStaticFiles** | ASP.NET 中间件，让 wwwroot 下的文件可以通过 URL 直接访问 |
| **GUID** | 全局唯一标识符，用来给上传的文件重命名，避免冲突 |
| **boundary** | multipart 请求中分隔不同"部分"的标记字符串，自动生成 |
