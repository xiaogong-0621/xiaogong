# 声域友线上社交KTV平台 — 项目说明书

---

## 一、项目概述

### 1.1 项目名称
**声域友（Hydro-Sonic Canvas / Azure Echo）** — 线上社交KTV点歌系统

### 1.2 项目定位
一套**线上社交KTV平台**，用户通过Web浏览器创建或加入虚拟房间，在房间内与朋友一起点歌、排队、实时聊天，享受线上KTV体验。管理端提供房间监控、歌曲库维护、用户管理和反馈处理功能。

### 1.3 项目性质
Web应用学科课程作业（团队项目），由线下KTV管理系统转型为线上社交KTV平台。

### 1.4 项目目标
- 将课程作业中的原型设计完整落地为可运行的全栈Web应用
- 管理端覆盖线上KTV运营所需的管理功能（房间监控、歌曲管理、用户管理、反馈处理）
- Web端覆盖用户在虚拟房间内的核心使用场景（注册登录、房间加入、选歌、点歌、排队、聊天、收藏）

### 1.5 项目范围
- **包含**：管理端8个页面 + Web端11个页面 + 微信小程序 + 后端RESTful API + SignalR实时通信 + 数据库设计
- **不包含**：支付/计费系统、移动端适配、多租户SaaS架构、视频通话

## 三、技术栈

### 3.1 技术栈总览

| 层级 | 技术选型 | 说明 |

| 后端框架 | ASP.NET Core 8.0 | RESTful API + SignalR Hub |

| Web构建工具 | Vite 8.0 | 快速HMR |
| Web UI框架 | Element Plus 2.13 + Tailwind CSS 3.4 | 组件库 + 原子化CSS |
| Web状态管理 | Pinia 3.0 | Vue 3官方推荐 |

| Web HTTP客户端 | Axios 1.15 | 请求/响应拦截器 |
| Web实时通信 | @microsoft/signalr 8.0 | SignalR客户端 |
| 小程序 | 微信小程序原生开发 | @babel/runtime 7.29 |
| 图标 | Material Symbols Outlined | 与原型一致 |
| 字体 | Plus Jakarta Sans + Manrope | 设计系统指定 |

### 3.2 架构模式
- **整体架构**：前后端分离B/S架构
- **后端分层**：Repository → Service → Controller 三层架构
- **实时通信**：SignalR（房间事件）+ REST轮询（聊天消息，2秒间隔）
- **认证方式**：JWT Bearer Token（无状态认证）
- **依赖注入**：Repository和Service均注册为Scoped生命周期

## 四、功能模块详解

### 4.1 管理端功能（8个页面）

#### 4.1.1 管理端登录
管理员通过用户名和密码登录管理后台。

#### 4.1.2 仪表盘（Dashboard）
- 4张统计卡片：在线房间数、在线用户数、今日创建房间数、总注册用户数
- 最新房间动态列表（最近5条活跃房间）
- 热门歌曲排行（Top 5，按累计播放量排序）
- 数据每3秒自动刷新

#### 4.1.3 房间监控
- **活跃房间列表**：查看所有虚拟房间（房间码、人数、状态、创建时间），支持按状态筛选和房间码搜索
- **开房申请审批**：审批用户的创建房间申请（通过/拒绝），通过后自动分配6位随机房间码
- **查看房间用户**：弹窗显示房间内所有在线用户
- **强制关闭房间**：踢出所有用户，房间码作废
- **空闲自动关闭**：所有用户退出后倒计时5分钟，期间有新用户加入则重置

#### 4.1.4 歌曲管理
- **统计卡片**：总曲目数、本周新增、今日播放量
- **歌曲列表**：分页展示，支持按歌名/歌手搜索、按流派筛选
- **新增歌曲**：填写歌曲信息 + 上传MP3文件（最大30MB）+ 上传封面（可选）+ 上传歌词（可选），自动从ID3标签读取时长和封面
- **歌曲详情页**：完整信息展示（封面、歌名、歌手、时长、播放量、收藏数、排名），支持在线编辑修改
- **软删除机制**：删除歌曲标记为disabled，不物理删除数据

#### 4.1.5 账户管理
- 用户列表：用户名、头像、状态（启用/禁用）、在线状态、所在房间、创建时间
- 支持搜索、筛选、新增账户、编辑信息、更改密码
- **禁用/启用用户**：禁用后用户无法登录，已在房间中会被踢出
- 在线状态：登录后10分钟内显示"在线"

#### 4.1.6 反馈管理
- 处理用户提交的歌曲相关反馈（请求添加歌曲/歌曲信息纠错/其他建议）
- 支持按状态筛选和搜索
- 通知图标显示待处理反馈角标数量

#### 4.1.7 系统设置
- **管理员账号**：修改用户名和密码（需密码验证）
- **基本信息**：平台名称、联系方式
- **安全设置**：日志保留期限、敏感操作二次验证（可配置4类操作独立开关）
- **操作日志**：记录所有管理员操作（登录、禁用用户、关闭房间、审批等），支持按类型/操作人/日期筛选，只读不可修改

#### 4.1.8 通知中心
查看和处理系统通知消息。

### 4.2 Web用户端功能（11个页面）

#### 4.2.1 注册
支持三种注册方式（任选其一）：
- 用户名 + 密码
- 手机号 + 密码
- 邮箱 + 密码

昵称为必填项。注册成功后自动跳转登录页。

#### 4.2.2 登录
输入账号（用户名/手机号/邮箱）和密码。登录成功后弹出房间选择窗口：
- 输入房间码加入已有房间
- 申请开房间（等待管理员审批）
- 稍后进入（关闭弹窗，先浏览歌曲）

登录成功返回JWT Token，存储到localStorage，过期自动跳转登录页。

#### 4.2.3 探索歌曲（核心页面）
- 流派筛选标签行（全部、流行、摇滚、民谣、电子、R&B、嘻哈、古典）
- 歌曲列表：封面、歌名、歌手、时长、收藏按钮、点歌按钮
- 悬浮预览效果（毛玻璃背景 + 阴影 + 微微放大）
- 右下角反馈浮动按钮（请求添加歌曲/纠错/建议）

#### 4.2.4 热门榜单
- 日榜/周榜切换
- Top 10排行：排名序号（第1名金色）、封面、歌名、歌手、播放量
- 每首歌都有点歌按钮
- 播放量格式化显示（超1万显示"x.x万"，超1千显示"xK"）

#### 4.2.5 我的收藏
- 展示用户收藏的歌曲列表
- 悬浮显示取消收藏按钮和点歌按钮
- 页面标题显示收藏数量

#### 4.2.6 当前房间（核心页面）
三区域布局：
- **房间信息栏**：房间码、在线人数、已点歌曲数、退出按钮
- **播放队列**：正在播放卡片（大封面、歌名、歌手、进度条、切歌按钮）+ 队列列表（序号、封面、点播者）
- **实时聊天**：消息列表 + 系统消息（进退房公告）+ 底部输入框，2秒轮询刷新

#### 4.2.7 底部播放器栏（全局组件）
- 底部居中浮动胶囊播放器（毛玻璃效果）
- 圆形封面 + 歌名歌手 + 内嵌进度条 + 三键控制（上一首/播放暂停/下一首）
- 点击封面展开全屏歌词页
- 空队列显示占位文字"未在播放"
- 使用全局单例HTML5 Audio实例，切页面不中断播放

#### 4.2.8 全屏歌词页
- 从底部滑入动画，封面模糊背景 + 毛玻璃遮罩
- 左侧：大封面、歌名、进度条、播放控制、播放模式切换、音量控制
- 右侧：漏斗式歌词滚动 + 播放列表侧栏（拖拽排序）
- 四种播放模式：顺序播放 → 列表循环 → 单曲循环 → 随机播放
- 歌词时间微调（±0.5s）、进度微调（±1s）
- 关闭方式：收起按钮 / Esc键

#### 4.2.9 用户主页
个人信息、统计数据、编辑资料、修改密码、最近点歌记录、深色模式切换。

#### 4.2.10 房间列表
搜索房间、申请开房、输入房间码加入。

#### 4.2.11 其他页面
- **消息通知页**：查看系统通知
- **隐私设置页**：隐私相关配置
- **关于页面**：使用说明（快速开始、核心功能、播放器指南、房间规则、技术栈）

### 5.2 关键设计要点
- 文本字段统一使用NVARCHAR（支持Unicode）
- 时间字段使用DATETIME2（高精度）
- 主键使用IDENTITY(1,1)自增
- 列名采用PascalCase命名
- 软删除策略（歌曲删除标记disabled，不物理删除）
- 外键约束确保数据完整性

## 六、实时通信架构

### 6.1 SignalR Hub（房间事件）
- 连接路径：`/hubs/ktv`
- 单个Hub处理所有房间生命周期事件
- 用户加入房间时自动加入SignalR Group `room-{roomId}`
- 服务端推送事件：UserJoined、UserLeft、QueueUpdated、RoomClosed、RoomApproved

### 6.2 REST轮询（聊天消息）
- 聊天消息通过REST API实现（每2秒轮询）
- 消息持久化到ChatMessages表
- 消息长度限制200字符
- 服务端敏感词过滤

### 6.3 播放状态同步
- 播放队列轮询（2秒间隔）
- 歌曲主模式：只有当前歌曲的点歌人可控制播放/暂停/切歌
- 四种播放模式生效：顺序播放、列表循环、单曲循环、随机播放
- History Stack管理播放历史，支持链式回退（上一首）

## 八、非功能性需求

### 8.1 性能
- 页面首次加载时间 < 3秒（本地开发环境）
- API接口响应时间 < 500ms
- 列表分页加载，单页不超过10条

### 8.2 兼容性
- 支持Chrome最新版本（主要开发和测试浏览器）
- 支持Edge最新版本

### 8.3 安全性
- 密码使用哈希算法存储，不明文存储
- API接口使用JWT Token认证
- 敏感操作需确认弹窗（管理员需密码二次验证）
- 聊天消息服务端敏感词过滤

### 8.4 可用性
- 所有交互操作提供视觉反馈（按钮按下缩放98%）
- 加载中状态显示骨架屏或加载指示器
- 操作结果使用消息提示（成功/失败）
- 响应式布局，内容区最小尺寸1024×600px

### 8.5 视觉设计
- **全局背景**：5个半透明浮动色块动画背景（CSS @keyframes），营造景深感
- **毛玻璃效果**：导航栏、播放器栏、卡片等关键组件使用backdrop-filter: blur()
- **悬浮预览**：歌曲列表行hover触发毛玻璃 + 阴影 + 微缩放效果
- **侧边栏收起**：256px ↔ 80px切换，300ms过渡动画
- **深色模式**：支持亮色/深色主题切换

### 8.6 代码质量
- 前端使用TypeScript严格模式
- 后端使用C#强类型
- API接口统一错误格式返回
- 全局异常处理中间件
